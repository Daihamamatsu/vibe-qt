# cline 開発ルール

以下は cline の開発における追加ルールです。

- **ソースコードの変更を行う場合**
  - まず GitHub Issue を作成し、変更内容と目的を明記します。
  - Issue に沿ったブランチ（例: `feature/<issue-id>-<short-description>`）を作成して作業します。
  - ブランチの変更が完了したら Pull Request (PR) を発行します。
  - PR のレビューを受け、マージされたことを確認して作業を完了とみなします。

これにより、変更履歴が追跡可能になり、コード品質とチーム協調性が向上します。
